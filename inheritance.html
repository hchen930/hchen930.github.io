Inheritance

Source: Blue Pelican Java by Charles E. Cook

Subclass and Superclass

BankAccount is an example of a superclass. SavingsAccount is an example of a subclass because it is a type of BankAccound and inherits all of BankAccount's properties and methods while being a unique and specific type of BankAccount. Subclasses can inherit from superclass by using the keyword "extends":
public class SavingsAcount extends BankAccount{}
Subclasses can construct a superclass object by using super(parameters) in its own constructor method.

Overriding and Shadowing

When the same method is declared twice in both the superclass and the subclass, objects made from the subclass will use the subclass method. Within subclass methods, overrident superclass methods can only be accessed using the super keyword again:
super.overridentMethodName(123);

Shadowing happens when state variables in both the superclass and the subclass share the same name. In this case, objects created from the subclass takes priority again. It is possible to still access the superclass variable by creating a getter method in the superclass:
public int getX () {return X;}
